## Context

本專案從零開始建立阿瓦隆線上遊戲。阿瓦隆是一款隱藏身份的多人推理桌遊，核心挑戰在於：部分資訊只有特定角色能看到（如梅林看得見壞人，但壞人不知道誰是梅林），因此必須由可信的伺服器端管理所有秘密資訊，不能將完整狀態暴露給前端。

技術限制：
- 不含帳號系統，玩家以 sessionStorage 中的 playerId 識別
- 不使用資料庫，所有遊戲狀態存於記憶體
- 前端無框架，純 HTML/CSS/JS

## Goals / Non-Goals

**Goals:**
- 伺服器為唯一的真相來源（single source of truth），所有遊戲邏輯在伺服器端驗證
- 角色秘密資訊只透過個人 WebSocket 推送，絕不廣播
- 完整遊戲流程：房間 → 角色分配 → 提名 → 投票 → 任務 → 刺殺
- 支援 5-10 人彈性人數，房主選擇角色組合

**Non-Goals:**
- 女士湖機制
- 帳號登入、歷史紀錄、持久化
- 觀戰模式、AI 玩家
- 手機 App（僅 Web 瀏覽器）

## Decisions

### 使用 ws 套件而非 Socket.IO

**決定**：使用原生 `ws` 套件而非 Socket.IO。

**理由**：Socket.IO 提供 fallback（long-polling）和自動重連，但會引入額外抽象層與較大套件體積。阿瓦隆是回合制遊戲，對連線中斷容忍度較高（重連後可重新同步遊戲狀態），不需要 Socket.IO 的複雜機制。`ws` 套件輕量且符合標準 WebSocket API。

**替代方案**：Socket.IO — 捨棄原因：額外複雜度不值得；SSE — 捨棄原因：單向通訊不足。

### 遊戲狀態機集中於 gameEngine.js

**決定**：所有 phase 轉換邏輯集中在 `server/gameEngine.js`，index.js 只負責訊息路由。

**理由**：遊戲邏輯與網路層分離，方便測試遊戲引擎；index.js 只做訊息解析與轉發，不含業務邏輯。

### 前端無框架

**決定**：前端用純 HTML/CSS/JS，不使用 React/Vue。

**理由**：遊戲介面狀態簡單（由伺服器完整推送 GAME_STATE），不需要 Virtual DOM 或響應式框架。減少建置複雜度，部署只需靜態檔案。

### 廣播 GAME_STATE 而非差異更新

**決定**：每次狀態變更後廣播完整的 GAME_STATE 物件（不含秘密資訊）。

**理由**：阿瓦隆是回合制，狀態變更頻率低，廣播完整狀態比差異更新簡單且不易出錯。前端只需根據 phase 字段渲染對應 UI，無需追蹤差異。

### 斷線處理策略

**決定**：玩家斷線時，若遊戲尚未開始則移除玩家；若遊戲進行中則保留座位，標記為 disconnected，等待重連（30 秒超時後判負）。

**理由**：開始前斷線不影響遊戲；開始後保留座位可避免因網路問題毀掉一局遊戲。

## Risks / Trade-offs

- **記憶體狀態**：伺服器重啟會清空所有房間。→ 接受此限制（MVP 階段）
- **無認證**：玩家可偽造其他人的 playerId。→ 使用 UUID v4 產生 playerId，難以猜測；非正式遊戲場景可接受
- **單一伺服器**：無水平擴展。→ 接受，MVP 不需要
- **R4 雙失敗規則**：容易忘記實作。→ 在 MISSION_REQUIREMENTS 表中明確標記，calcMissionResult() 接受 roundIndex 參數
